# Day 04 – Linux Practice: Processes and Services  


On **Day 04**, I practiced basic Linux commands for processes, services, and logs.  
The goal was simple: use real commands and understand how troubleshooting works.

---

## What I Practiced  

### 🔹 Process Check  
- Used `ps aux` and `top` to see running processes  
- Checked CPU and memory usage  

### 🔹 Service Check  
- Used `systemctl status` to check a service  
- Viewed active services  

### 🔹 Logs  
- Used `journalctl` to see logs  
- Used `tail` to check recent logs  

---

## Key Learning  

Processes, services, and logs are connected.  
Understanding them helps in troubleshooting Linux systems.
